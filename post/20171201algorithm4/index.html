1.2 背包、队列、栈

背包是这样的一种数据结构：只支持添加，不允许删除元素，并且遍历其中元素的顺序是不确定的。

这三种数据结构都可以用 数组 或者 链表 来实现。

若使用数组实现，则当数组空间被占满后，需要将其扩容，即创建一个容量更大的新数组，然后将老数组中的数组拷贝到新数组中。数组还有一个缺点，就是当在数组的非尾部增删元素时，会涉及大量元素的移动操作，因此很慢。

链表的缺点是占用空间比数组大；不能像数组那样直接通过索引定位到中间的某一个元素。优点是在任意位置增删元素都很快。

java中提供了Stack类来作为栈。但Queue只是一个接口，LinkedList实现了Queue接口，可以当做队列来使用。

1.3 动态连通性问题（union-find算法）▲

动态连通性问题：使用一个数据结构来存储大量的对象，对于给定的任意两个对象，此数据结构都能迅速告诉用户：给定的两个对象是否是连通的。所谓“动态”指的是，连通性不是一成不变的，比如，我们可能会经常在两个对象之间添加一条连接。

为了方便，我们将所有对象(又称为触点、节点、顶点等)以从0开始的正整数命名。而分量以分量中的某一个触点命名。

我们需要实现以下几个API：

  • void union(int p,int q):在p和q之间添加一条连接
  • int find(int p):找出p所在的分量（相互连通的一堆节点合称为一个分量）
  • boolean connected(int p,int q):判断p和q是否连通。这个方法的实现很简单，只要“return find(p)==find(q);”即可。

1.3.1 quick-find算法

创建了一个数组 int[] id，数组的索引是触点名，值是该触点所在的分量。

find的实现很简单，只要“return id[p];”即可。

而对于union操作，则需要遍历整个数组，将所有值为id[q]的元素赋值为为id[p]（或者将所有值为id[p]的元素赋值为为id[q]）。

1.3.2 quick-union算法

数组 int[] id的索引是触点，而值则是该触点所在分量之中的另一个触点(将这种关系称为链接)。

在实现find()方法时，我们从给定的触点开始，由它找到另一个触点，再由这个触点找到第三个触点，以此类推，直到找到根触点(链接到自己的触点，即id[m] == m)为止。此分量也以根触点命名，即m。

union(int p,int q)的实现也很简单，只要分别找到p和q的根触点，然后将一个根触点链接到另一个即可。

此外，为了改善quick-union在最坏情况下的性能，还有“加权quick-union算法”，即每次union操作时，总是将较小的树连接到较大的树上。

1.3.3 动态连通性问题的应用场景

  • 两台计算机是否在同一个网络中
  • 电路中的两个触点是否连通
  • 社交网络
  • 变量名的等价性
  • ……

原地排序和非原地排序

除了函数调用所需要的栈和固定数目的实例变量之外，无需额外内存空间来存储另一份数组副本 的排序算法称为原地的。

后面要介绍的几种排序算法，除了归并排序是非原地的，其他的几个都是原地的。

排序的稳定性

如果一个排序算法 能够保留数组中重复元素的相对位置 则可以被称为是稳定的。例如，obj1、obj2、obj3的key(排序的依据)都是5，如果在排序之后，这三个元素的顺序依然是obj1、obj2、obj3，则此排序算法是稳定的。

稳定性是一个很重要的性质。例如，设想有一种event对象，它有时间和地点这两个key。首先我们将一系列的event按时间先后排序。接着，我们希望将这些event按发生的地点进行分组，如果分组使用的是稳定的排序算法，那么在分组之后，同一组内的event依然是按照时间先后顺序排列的；但如果分组使用的是非稳定的排序算法，那么分组后同一组内的event极有可能是时间乱序的。

后面要介绍几种排序算法，只有插入排序和归并排序是稳定的。

//将有序的子数组a[lo..mid]与a[mid+1..hi]归并成一个有序的数组，并将结果存放在a[lo..hi]中
//会先将涉及的所有元素复制到一个辅助数组中，再把归并的结果放回原数组中
public static void merge(Comparable[] a, int lo, int mid, int hi) {
    //将a[lo..hi]复制到aux[lo..hi]
    for (int k = lo; k <= hi; k++) {
        aux[k] = a[k];
    }

    // 初始时，指针指向两个子数组的第一个元素
    int i = lo;
    int j = mid + 1;

    //归并
    //遍历a[lo..hi],决定每一个a[k]应该被赋值为aux[i]还是aux[j]
    for (int k = lo; k <= hi; k++) {
        if (i > mid) {//第一个子数组已经归并完毕
            a[k] = aux[j++];
        } else if (j > hi) {//第二个子数组已经归并完毕
            a[k] = aux[i++];
        //以下两个分支为正常的逻辑(两个子数组都未归并完毕)
        } else if (less(aux[j], aux[i])) {
            a[k] = aux[j++];
        } else {
            a[k] = aux[i++];
        }
    }
}

2 自底向上的归并排序(了解)

通常使用递归操作来完成的逻辑都能改用多层循环来实现，只要换一个方向。递归虽然容易想到也容易理解，但是会造成函数调用栈中同时存在大量的函数，使内存占用快速增涨。

先归并那些微型数组，然后再成对归并得到的子数组，如此这般，直到我们将整个数组归并到一起。首先我们进行的是两两归并(把每个元素想象成一个大小为1的数组)，然后是四四归并(把两个大小为2的数组归并成一个大小为4的数组)，然后是八八归并，一直下去。

public static void sort(Comparable[] a) {
    int N = a.length;

    //外层循环控制子数组的大小，初始为1，然后每次循环都加倍
    for (int sz = 1; sz < N; sz += sz) {
        //内层循环从左到右将子数组两两归并
        //若lo >= N - sz，说明只剩下一个子数组了，则不再继续两两归并
        for (int lo = 0; lo < N - sz; lo += 2sz) {
            merge(a, lo, lo + sz - 1, Math.min(lo + 2sz - 1, N - 1));
        }
    }
}

参与归并的两个数组角标：[lo, lo + sz - 1]、[lo+sz, Math.min(lo + 2sz - 1, N - 1)]。

若lo>=N-sz，即lo+sz>=N，即lo+sz-1>=N-1，也就是说，余下元素只够一个数组了，也就无法继续归并了，因此内层循环条件是lo<N-sz。

2.5 快速排序▲

基本思想

快速排序和归并排序一样，是一种 “分治” 的排序算法。

在归并排序中，一个数组（被大致地）等分为两半；在快速排序中，切分(partition)的位置取决于数组的内容，具体规则如下：

  1. 对于某个j,a[j]已经排定
  2. a[lo]到a[j-1]中的所有元素都<=a[j]
  3. a[j+1]到a[hi]中的所有元素都>=a[j]

归并排序将数组分成两个子数组分别排序，并将有序的子数组归并以将整个数组排序；而快速排序将数组切分为两个子数组，只要将两个子数组排序好，那么整个数组也就有序了。

切分的实现

通常是先取a[lo]作为切分元素，即那个将会被排定的元素，然后我们从数组的左端(lo+1)开始向右扫描直到找到一个大于等于它的元素，再从数组的右端(hi)开始向左扫描直到找到一个小于等于它的元素，交换这两个元素的位置。如此继续，我们就可以保证左指针i左侧的元素都<=切分元素，右指针j右侧的元素都>=切分元素。当两个指针相遇或交叉时（此时a[j]必然小于等于a[lo]），我们只要将切分元素a[lo]和a[j]交换然后返回j即可。详见p184图。

//将数组切分为a[lo..j-1],a[j],a[j+1..hi]，返回切分点j
private static int partition(int[] a, int lo, int hi) {
    int v = a[lo];//切分元素
    int i = lo, j = hi + 1;//左右扫描指针。刚开始位于起点之外

    // 找到目标i、j，并交换对应的数组元素
    // 若i、j相遇或交叉，则退出循环
    while (true) {
        while (a[++i] < v) {//找到>=v的元素
            if (i == hi) break;
        }
        while (a[--j] > v) {//找到<=v的元素
            if (j == lo) break;
        }

        if (i < j) {
            exch(a, i, j);//交换上面找到的两个元素
        } else {//如果指针相遇或交叉，则退出while(true)循环
            break;
        }
    }

    exch(a, lo, j);//交换切分元素和a[j]
    return j;
}

性能

与切分的状况有关，大部分情况下时间复杂度为NlogN，最坏的情况下时间复杂度为N2。

最坏的情况例如第一次从最小的元素切分，第二次从第二小的元素切分……我们要在快速排序前将数组打乱的主要原因就是避免这种情况，它能够使产生糟糕切分的可能性降到最低，我们就无需为此担心了。

快速排序切分方法的内循环会用一个递增的索引将数组元素和一个定值比较，这种简洁性是快速排序的一个优点，很难想象排序算法还能有比这更短小的内循环了，归并排序一般都比快速排序慢，其原因就是它还在内循环中移动数据。

此外，快速排序相比归并排序还有一个优点，就是快速排序是原地排序算法。

如前面归并排序中所说，对于较小规模的数组(如长度<15)，插入排序的速度可能更快。因此可以考虑当子数组的长度<15时切换到插入排序，以此来提高快速排序的整体性能。

2.6 三向切分的快速排序（了解）

基本思想

实际应用中经常会出现含有大量重复元素的数组，对于这些数组，将快速排序的切分方法稍加改进就能将性能从NlogN级别提升到N级别。

一个简单的想法是每次将数组切分为3部分，分别对应小于、等于、大于切分元素的数组元素。具体做法为从左到右遍历数组一次，维护一个指针lt使得a[lo..lt-1]中的元素都小于v，一个指针gt使得a[gt+1,hi]中的元素都大于v，一个指针i使得a[lt..i-1]中的元素都等于v，a[i..gt]中的元素都还未确定，详见p299图2.3.4。我们使用Comparable接口(而非less方法)来对a[i]和v进行三项比较：

  • a[i]<v，将a[lt]与a[i]交换，将lt和i加1；
  • a[i]>v，将a[gt]与a[i]交换，将gt减1；
  • a[i]==v,将i加1。

public class Quick {

        StdRandom.shuffle(a);//打乱顺序，消除对输入的依赖
        sort(a, 0, a.length - 1);
    }

    private static void sort(Comparable[] a, int lo, int hi) {
        if (hi <= lo) return;//递归结束

        //切分开始--------------------
        int lt = lo, i = lo + 1, gt = hi;
        Comparable v = a[lo];//切分元素
        while (i <= gt) {
            int cmp = a[i].compareTo(v);
            if (cmp < 0) {
                exch(a, lt++, i++);
            } else if (cmp > 0) {
                exch(a, i, gt--);
            } else {
                i++;
            }
        }
        //切分结束--------------------

        sort(a, lo, lt - 1);
        sort(a, gt + 1, hi);
    }
}

这样的切分方式能够将与切分元素相等的元素归位，这样它们就不会被包含在递归调用处理的子数组中了。对于存在大量重复元素的数组，这种方法比标准的快速排序要高效很多。

性能

对于不含大量重复元素的数组，比标准快速排序略慢，因为切分时有更多次交换。

对于含有大量重复元素的数组可将标准快速排序的性能由NlogN级别提升到N级别。

2.7 优先队列▲

2.7.1 概念

很多情况下我们会在集合中收集一些元素，处理并删除集合中最大的元素，然后再添加进一些元素，再处理并删除集合中最大的元素，如此这般。

在这种情况下，一个合适的数据结构应该支持两种操作：1删除并返回最大元素、2插入元素。这种数据结构叫做优先队列（priority queue）(此处是最大优先队列，最小优先队列也是一样的道理)。

java系统库提供的优先队列见java.util.PriorityQueue。

2.7.2 堆

概念

二叉堆 是 用以表示完全二叉树 的一种数据结构。通常是一个数组。

完全二叉树：设二叉树的深度为h，除第 h 层外，其它各层 (1～h-1) 的结点数都达到最大个数，第 h 层所有的结点都连续集中在最左边，这就是完全二叉树。

完全二叉树只用数组而不需要指针就可以表示，具体方法就是将二叉树的结点按照层级顺序放入数组中，根结点在位置1(位置0空着不用)，它的子节点在位置2和3，而子节点的子节点则分别在位置4、5、6和7，以此类推。

一棵大小为N的完全二叉树的高度为log2N。

当一棵二叉树的每个结点都大于等于它的两个子结点时，称为 堆有序的二叉树。很明显，根结点是堆有序的二叉树中的最大结点。

二叉堆一般用数组来表示，从位置1开始存储(位置0空着不用)。位置k的结点的父结点的位置为k/2，而它的两个子结点的位置则分别为2k和2k+1。这样在不使用指针的情况下我们也可以通过计算数组的索引在树中快速地上下移动。

2.7.4 堆排序

算法：先将原始数组重新组织安排进一个堆实现的最小优先队列中，再从优先队列中逐个删除最小元素。

堆排序的时间复杂度平均为NlogN，但总体来说比归并和快速排序要慢，因此较少使用，了解思想即可。

2.8 总结

排序如此有用的一个主要原因是，在一个有序数组中查找一个元素要比在一个无序的数组中容易得多。

实际使用中常用的排序算法有3种：插入排序、归并排序、和快速排序(包括3向切分的快速排序)。

  • 插入排序：对于部分有序的数组，效率极高，对于小规模数组(如长度<15)速度也较快。稳定、原地。
  • 归并排序：速度与快速排序基本相当。稳定、非原地。
  • 快速排序：快速排序是速度最快的通用排序算法。非稳定、原地。

java.util.Arrays.sort()，对于原始数据类型元素的数组使用三向切分的快速排序，对于引用型元素的数组使用归并排序。

java.util.Collections.sort()是稳定的(官方文档：This sort is guaranteed to be stable: equal elements will not be reordered as a result of the sort)，所以应该也是归并排序。

两个典型的应用：

  • 找出重复元素

    首先将数组排序，然后遍历有序的数组，记录连续出现的重复元素即可。

  • 负载均衡

    假设我们有M个相同的处理器以及N个任务，我们的目标是用尽可能短的时间完成所有任务。这个问题是NP-困难的(见第6章)，因此我们实际上不可能算出一种最优的方案。但我们可以使用这样一种较优的方案：为M个处理器维护一个最小优先队列，每个处理器的key就是此处理器上运行的任务的耗时之和。每次我们都取出key最小的处理器，将下一个任务分配给这个处理器，然后将它重新插入优先队列。

3 查找

学习目标：知道红黑树和散列表即可，其他的都是铺垫。

符号表

符号表是一种存储键值对的数据结构。支持两种操作：插入(put)，即将一个新的键值对存入表中；查找(get)，即根据给定的键得到相应的值。

几个规定

为了简化问题，通常的符号表都有下面几个规定：

  1. 每个键只对应着一个值(即表中不允许存在重复的键)，当向表中存入一个键值对时，如果键已经存在，那么新的值会覆盖旧的值。
  2. 键和值都不能为null

有序符号表和无序符号表

这里有序符号表是指会保持键的大小顺序，相比无序符号表，有序符号表除了提供基本的put()、get()操作外，还提供min()、max()、floor()、ceiling()、rank()、select()等与顺序有关的操作。

典型的有序符号表比如java中的TreeMap(底层是红黑树)；典型的无序符号表比如java中的HashMap(底层是基于拉链法的散列表)。

3.1 两种最简单的符号表实现

3.1.1 无序链表

键值对不是按照键的大小顺序来存储的。查找需要遍历链表，插入也需要遍历链表(因为需要先进行查找，然后根据键是否已存在决定是覆盖还是插入)，因此查找和插入的时间复杂度都是N。

3.1.2 有序数组

键值对按照键的大小顺序存储在数组中(使用一对平行的数组，一个存储键，一个存储值)。 查找的时间复杂度为logN(使用二分查找)。

插入的时间复杂度为N(将键值对插入到数组中的某个位置，参考插入排序，涉及大量的数组元素的移动)。

3.2 二叉查找树▲

二叉查找树是一棵二叉树，每个结点的键都大于其左子树中任意结点的键而小于右子树中任意结点的键(前面规定了键不能重复，因此不存在相等的情况)。

二叉查找树的形状和高度与键值对被插入的顺序有关，最好的情况下，二叉树是平衡的，其高度为log2N，而最差的情况下(键值对被按照键从大到小或从小到大的顺序插入)，其高度为N。

在二叉查找树中插入元素：从根节点开始往下查找，如果没有找到，也就是在前进的路径上遇到了空的分叉，那就将被插入元素放在这个空分叉上。

二叉查找树的插入和查找操作的时间复杂度取决于树的高度(因为这些操作最主要的过程就是沿着树的高度方向进行遍历)，因此二叉查找树的插入和查找操作的时间复杂度，通常的情况下为logN，但最坏的情况下为N。

3.3 平衡查找树▲

2-3查找树

2-3查找树中有2-和3-两种结点，通过特有的操作方式，我们可以保证在进行任何操作之后，2-3查找树都是完美平衡的。

简洁起见，这里我们用2-3查找树指代一棵完美平衡的2-3查找树(在其他的情况下这个词应该表示一种更一般的结构)。

当不断的插入元素时，二叉查找树由上往下生长，而2-3查找树是由下往上生长的。

因为2-3查找树是完美平衡的，因此其查找和插入操作的时间复杂度都是logN。

红黑树

红黑树是2-3查找树的另一种表现形式，它本质上和2-3查找树是完全等价的。因此，红黑树的查找和插入操作的时间复杂度都是logN。

3.4 散列表▲

通过散列函数将键转化为数组的索引来访问数组中的键值对。设计一个散列表主要考虑两点：1.散列函数的实现2.处理碰撞。

使用散列表，你可以实现在一般的应用中拥有常数级别的查找和插入操作的符号表。相比平衡查找树，散列表的查找、插入性能更优，但平衡查找树中的元素是有序的，而散列表是无序的，因此散列表不支持有序性相关的操作。

3.4.1 散列函数的实现

对于不同类型的键，散列函数有不同的设计思想，详细参考书p294。将整数类型的键散列的最常用方法是除留余数法：我们选择大小为素数M的数组，对于任意正整数k，计算k%M，就能有效的将键散布在0到M-1的范围内。如果M不是素数，我们可能无法利用键中包含的所有信息，这可能导致我们无法均匀的散列。

在java中，对于基本数据类型和常用的类，系统已经提供了可靠的散列函数，即hashCode()。

3.4.2 处理碰撞

拉链法

将大小为M的数组中的每个元素指向一条链表，链表中的每个结点都存储了被散列到此数组索引的所有键值对。

在拉链法的散列表中查找分为两步：1根据散列值找到对应的链表2沿着链表顺序查找相应的键。

在实现基于拉链法的散列表时，我们的目标是选择适当的数组大小M，既不会因为空链表而浪费大量内存，也不会因为链表太长而在第二步遍历查找中花费太多时间。

3.5 总结

public class DepthFirstSearch {
    private boolean[] marked;    // 某顶点是否与s连通(即搜索过程中是否到达过此顶点)
    private int count;           // 与s连通的顶点数量

        dfs(G, s);
    }

            }
        }
    }

    //某顶点是否与s连通
    public boolean marked(int v) {
        return marked[v];
    }

    public int count() {
        return count;
    }

4.1.7 广度优先搜索（寻找最短路径）▲

通过DFS找到的是从s到v的多条路径中的一条，它不一定是最短的。

要寻找最短路径，经典的方法是广度优先搜索(BFS，Breadth First Search)。

基本思路：要找到从s到v的最短路径，从s开始，先在所有由一条边就可以到达的顶点中寻找v，如果找不到就继续在与s距离两条边的所有顶点中查找v，如此一直进行。

技巧：使用一个队列来保存所有已经被访问过，但其邻接表还未被访问过的顶点。每次从队首取出一个点，访问它邻接表中的所有点，然后将这些点全部放入队尾，然后再从队首取出一个点，访问它邻接表中的所有点，然后将这些点全部放入队尾……以此类推。

队列其实就是用来解决按照顺序进行一系列操作这样的问题。

    public BreadthFirstPaths(Graph G, int s) {
        marked = new boolean[G.V()];

        Queue<Integer> queue = new Queue<Integer>();
        marked[s] = true;
        queue.enqueue(s);

        while (!queue.isEmpty()) {
            int v = queue.dequeue();
            for (int w : G.adj(v)) {
                if (!marked[w]) {
                    edgeTo[w] = v;
                    marked[w] = true;
                    queue.enqueue(w);
                }
            }
        }
    }

        return marked[v];
    }

        //和深度优先搜索中的实现相同
    }

4.2 有向图

4.2.1 概念

关于概念不要纠结，了解大概即可。

和无向图一样，本节也不考虑自环和平行边(即重复边：连接相同的两个顶点，且方向相同的边)。

4.2.4 环与拓扑排序▲

优先级限制下的调度问题：设想大学时期的选课，有50门课程，其中一些课程需要先修完它的前置课程才能修习，那么应该如何规划这50门课程的修习顺序？更一般点的描述是：给定一组需要完成的任务，以及关于任务完成的先后次序的优先级限制，那么在满足限制条件的前提下应该如何安排并完成所有任务？解决上述问题的办法就是拓扑排序。

拓扑排序：给定一幅有向图，将所有的顶点排序，使得所有的有向边均从排在前面的元素指向排在后面的元素。

public class DirectedCycle {
    private boolean[] marked;
    private int[] edgeTo;
    private boolean[] onStack;//递归调用栈中的顶点。如果某个顶点在栈中，说明它在本次搜索的路径中
    private Stack<Integer> cycle;//有向环

    public DirectedCycle(Digraph G) {
        marked = new boolean[G.V()];
        onStack = new boolean[G.V()];
        edgeTo = new int[G.V()];
        for (int v = 0; v < G.V(); v++){
            if (!marked[v] && cycle == null) dfs(G, v);
        }
    }

    //此dfs的目的是寻找有向环，找到了就不再继续
    private void dfs(Digraph G, int v) {
        //方法调用开始，将v加入onStack[]
        onStack[v] = true;
        marked[v] = true;
        for (int w : G.adj(v)) {
            if (cycle != null) {
                return;
            } else if (!marked[w]) {
                edgeTo[w] = v;
                dfs(G, w);
            //w已被访问过，且在栈中(即在本次查找的路径上)，说明找到了环
            //单单w被访问过无法说明问题，w可能是本条路径中的前面的顶点，也可能是另一条路径中的顶点，如上图中的顶点D
            } else if (onStack[w]) {
                cycle = new Stack<Integer>();
                //借助上面的图去理解此处的逻辑
                for (int x = v; x != w; x = edgeTo[x]) {
                    cycle.push(x);
                }
                cycle.push(w);
                cycle.push(v);
            }
        }
        //方法调用结束，则将v从onStack[]中移除
        //清空了标记位，这样，当一条路径搜索完毕之后，onStack[]中所有元素都是false
        onStack[v] = false;
    }

    public boolean hasCycle() {
        return cycle != null;
    }

    public Iterable<Integer> cycle() {
        return cycle;
    }
}

public class DepthFirstOrder {
    private boolean[] marked;
    private Queue<Integer> pre;//前序
    private Queue<Integer> post;//后序
    private Stack<Integer> reversePost;//逆后序

    public DepthFirstOrder(Digraph G) {
        pre = new Queue<Integer>();
        post = new Queue<Integer>();
        reversePost = new Stack<Integer>();

        marked = new boolean[G.V()];
        for (int v = 0; v < G.V(); v++)
            if (!marked[v]) dfs(G, v);
    }

    private void dfs(Digraph G, int v) {
        pre.enqueue(v);//前序

        marked[v] = true;
        for (int w : G.adj(v)) {
            if (!marked[w]) {
                dfs(G, w);
            }
        }

        post.enqueue(v);//后序
        reversePost.push(v);//逆后序
    }

    public Iterable<Integer> pre(){
        return pre;
    }

    public Iterable<Integer> post(){
        return post;
    }

    public Iterable<Integer> reversePost(){
        return reversePost;
    }
}

4.2.5 强连通性

如果两个顶点v和w是相互可达的，则称它们是强连通的。如果一幅有向图中的任意两个顶点都是强连通的，则称该有向图是强连通的。

两个顶点是强连通的，当且仅当它们都在一个有向环中。

设计一种平方级别的算法来计算强连通分量并不困难，但这在处理大规模问题时是无法接受的。

  1. 在有向图G中，使用DepthFirstOrder来计算它的反向图GR的逆后序排列。

  3. 在构造函数中，所有在同一个递归dfs()调用中被访问到的顶点都在同一个强连通分量中。

    public KosarajuCC(Digraph G) {

        DepthFirstOrder order = new DepthFirstOrder(G.reverse());
        for (int s : order.reversePost()) {
            if (!marked[s]) {
                dfs(G, s);
                count++;
            }
        }
    }

        marked[v] = true;
        id[v] = count;
        for (int w : G.adj(v)) {
            if (!marked[w]) {
                dfs(G, w);
            }
        }
    }

    public boolean stronglyConnected(int v, int w) {
        return id[v] == id[w];
    }

    public int id(int v) {
        return id[v];
    }

    public int count() {
        return count;
    }

4.3 加权无向图 & 最小生成树（了解）

4.3.1 概念

图的生成树：一棵含有其全部顶点的无环连通子图。最小生成树即所有边权值最小的生成树。

为了简化问题有如下设定：

  1. 假设图是连通图
  2. 边的权重可能是0或者负数
  3. 所有边的权重都各不相同

树的两个重要性质（无向图）：

  1. 用一条边连接树中的任意两个顶点都会产生一个环
  2. 从树中删去一条边将会得到两棵独立的树

public class Edge implements Comparable<Edge> {

    private final int v;
    private final int w;
    private final double weight;

    public Edge(int v, int w, double weight) {
        this.v = v;
        this.w = w;
        this.weight = weight;
    }

    public double weight() {
        return weight;
    }

    public int either() {
        return v;
    }

    public int other(int vertex) {
        if (vertex == v) return w;
        else if (vertex == w) return v;
        else throw new IllegalArgumentException("Illegal endpoint");
    }

    @Override
    public int compareTo(Edge that) {
        return Double.compare(this.weight, that.weight);
    }

    public String toString() {
        return String.format("%d-%d %.5f", v, w, weight);
    }
}

    }

    public void addEdge(Edge e) {
        int v = e.either();
        int w = e.other(v);
        adj[v].add(e);
        adj[w].add(e);
        E++;
    }

    public Iterable<Edge> adj(int v) {
        return adj[v];
    }

    //返回所有边，略
    public Iterable<Edge> edges() {
    }
}

4.3.3 求最小生成树的原理

切分：将图的所有顶点分为两个非空且不重叠的集合。

横切边：横跨两个集合的边。

切分定理：在一幅加权图中，给定任意的切分，它的横切边中的权重最小者必然属于图的最小生成树。

贪心算法：1.将属于最小生成树的边标记为黑色，初始状态下所有边均为灰色。2.找到一种切分，它产生的横切边均不是黑色，将它权重最小的横切边标记为黑色，如此反复，直到标记了V-1条黑色边为止。

public class LazyPrimMST {
    private boolean[] marked;//最小生成树中的顶点
    private Queue<Edge> mst;//最小生成树中的边
    private MinPQ<Edge> pq;//所有横切边(包括已失效的横切边)
    //若一条横切边中的两个顶点都在最小生成树中，那么它就已经不是横切边了，称为“失效”
    //由visit()方法可知，当横切边被加入pq时，它们都是有效的，但随着运算的进行，队列中的横切边可能失效（参考书p399 图4.3.10）

    public LazyPrimMST(EdgeWeightedGraph G) {
        marked = new boolean[G.V()];
        mst = new Queue<Edge>();
        pq = new MinPQ<Edge>();

        visit(G, 0);
        while (!pq.isEmpty()) {
            Edge e = pq.delMin();//取出权重最小的横切边
            int v = e.either(), w = e.other(v);
            if (marked[v] && marked[w]) continue;//跳过失效的横切边
            mst.enqueue(e);//将横切边添加到树中

            //将横切边不在树中的那一个顶点加入树中，并将与此顶点相邻的所有横切边加入最小优先队列pq
            if (!marked[v]) visit(G, v);
            if (!marked[w]) visit(G, w);
        }
    }


    //将v加入生成树，并将与v相连的所有横切边加入最小优先队列pq
    private void visit(EdgeWeightedGraph G, int v) {
        marked[v] = true;
        for (Edge e : G.adj(v))
            if (!marked[e.other(v)]) pq.insert(e);//另一点不在树中，说明是横切边
    }

    public Iterable<Edge> edges() {
        return mst;
    }

    public double weight() {
        //略
    }
}

延时Prim算法计算一幅连通加权无向图的最小生成树所需时间与ElogE成正比(最坏情况)，所需空间与E成正比。

public class PrimMST {
    private boolean[] marked;//最小生成树中的顶点
    private Edge[] edgeTo;//edgeTo[v]:将v连接到树中的权重最小的边，也就是最终的最小生成树中的边
    private double[] distTo;// distTo[v]:最小的权重(将v连接到树中的权重最小的边的权重)
    private IndexMinPQ<Double> pq;//这是一个索引优先队列，索引为顶点(如v)，值为该顶点的最小权重即distTo[v]

    public PrimMST(EdgeWeightedGraph G) {
        marked = new boolean[G.V()];
        edgeTo = new Edge[G.V()];
        distTo = new double[G.V()];
        pq = new IndexMinPQ<Double>(G.V());
        //将权重初始化为正无穷
        for (int v = 0; v < G.V(); v++)
            distTo[v] = Double.POSITIVE_INFINITY;

        distTo[0] = 0.0;
        pq.insert(0, 0.0);
        while (!pq.isEmpty()) {
            visit(G, pq.delMin());
        }
    }

    private void visit(EdgeWeightedGraph G, int v) {
        //将v加入生成树
        marked[v] = true;

        //与lazyPrimMST对比
        for (Edge e : G.adj(v)) {
            int w = e.other(v);
            if (!marked[w]) {
                //e是一条横切边
                //lazyPrimMST是将e直接加入最小优先队列
                //而这里还要进一步筛选
                if (e.weight() < distTo[w]) {
                    distTo[w] = e.weight();
                    edgeTo[w] = e;
                    if (pq.contains(w)) {
                        pq.change(w, distTo[w]);//替换
                    } else {
                        pq.insert(w, distTo[w]);//新增
                    }
                }
            }
        }
    }

    public Iterable<Edge> edges() {
    }

    public double weight() {
    }
}

4.4.6 Kruskal算法

Kruskal算法的思想很简单：将图中所有的边加入最小优先队列，每次取出权重最小的边将其加入最小生成树中，确保它不会和树中已存在的边形成环，如此继续，直到树中含有V-1条边为止。

public class KruskalMST {
    private Queue<Edge> mst;

    public KruskalMST(EdgeWeightedGraph G) {
        mst = new Queue<Edge>();
        MinPQ<Edge> pq = new MinPQ<Edge>();
        //将所有边插入优先队列
        for (Edge e : G.edges()) {
            pq.insert(e);
        }
        //union-find数据结构，用来识别会形成环的边
        UF uf = new UF(G.V());

        while (!pq.isEmpty() && mst.size() < G.V() - 1) {
            Edge e = pq.delMin();
            int v = e.either();
            int w = e.other(v);
            //跳过加入树中后会形成环的边
            if (!uf.connected(v, w)) {
                uf.union(v, w);
                mst.enqueue(e);
            }
        }
    }

    public Iterable<Edge> edges() {
        return mst;
    }

    public double weight() {
    }
}

int N = a.length;
String[] aux = new String[N];//辅助数组
int[] count = new int[R+1];

//1.遍历数据对象，统计每个键出现的次数
//将键i出现的次数保存在count[i+1]中(这样做是为了方便后面第3步中使用)
for(int i=0;i<N;i++){
    count[a[i].key + 1]++;
}

//2.从左往右对count[]数组的每个元素进行累加
//这样的结果就是:count[i+1]等于键0到i出现的次数之和，也就是键为i+1的数据对象在排序结果数组中的起始索引
for(int r=0;r<R;r++){
    count[r+1] += count[r]; 
}

//3.遍历数据对象，将它们按照索引依次复制到数组aux[]中，排序完成
for(int i=0;i<N;i++){
    aux[count[a[i].key]++] = a[i];
}

//回写
for(int i=0;i<N;i++){
    a[i] = aux[i];
}

键索引计数法排序N个键为0~R-1之间的整数的元素需要访问数组11N+4R+1次，这突破了之前的排序算法NlogN的时间复杂度下限。

键相同的元素在排序后会被聚集到一起，但相对位置没有变化，即这种排序算法是稳定的。

5.1.2 低位优先的字符串排序▲

即从右往左检查字符串中的字符。

通过字母表，我们可以将一个字母转化为它在字母表中的索引，也就是一个0到R-1之间的整数(R为字母表中的字母个数)。

算法思想：如果字符串的长度均为W，那就从右向左以每个位置的字符作为键，用键索引计数法将字符串排序W遍，就可以得到正确的排序结果。

算法实现：见p459 算法5.1，很简单。

时间复杂度：对于基于R个字符的字母表的N个长度为W的字符串，低位优先的字符串排序需要访问~7WN+3WR次数组，这可以认为是一种线性的排序算法。

5.1.3 高位优先的字符串排序▲

即从左往右检查字符串中的字符。

算法思想：首先用键索引计数法将所有字符串按首字母排序，对于拥有相同首字母的字符串子集，我们再将其按照第二个字母进行排序……

时间复杂度：对于基于R个字符的字母表的N个长度为W的字符串，高位优先的字符串排序需要访问8N+3R到~7WN+3WR次数组。

5.2 单词查找树▲

5.2.1 R向单词查找树

利用字符串的性质我们可以开发出比第3章中介绍的通用算法效率更高的查找算法，在一般的应用场景中它能取得以下的性能：1.查找命中所需的时间与被查找的键的长度成正比；2.查找未命中平均只需检查几个字符。

缺点：占用空间大。

单词查找树的结构

  • 每个节点都只可能有一个指向它的节点，称为它的父节点（根节点除外，根节点没有父节点）。
  • 每个节点都含有R条连接（R为字母表的大小），如果一条连接没有指向另一个节点，则称之为空连接，在绘制一棵单词查找树时通常会忽略空连接。
  • 每个节点也都含有一个值，可以是空也可以是符号表中的某个键所关联的值。具体来说，我们将每个键所关联的值保存在该键的最后一个字母所对应的节点中。

单词查找树中的查找操作

从根节点开始，首先经过的是单词首字符所对应的连接；在下一个节点处沿着单词第二个字符所对应的连接继续前进；在第二个节点中沿着单词第三个字符所对应的连接向前……如此这般，直到到达了单词最后一个字符所对应的节点或是遇到了一条空连接（即没有指向任何节点的连接）：

  1. 单词的尾字符所对应的节点中的值非空——这是一次命中的查找。
  2. 单词的尾字符所对应的节点中的值为空——这是一次未命中的查找。
  3. 查找结束于一条空连接——这也是一次未命中的查找。

5.2.2 三向单词查找树

R向单词查找树能实现极高的查找效率，但因为每个节点中都含有R条连接，因此空间占用十分巨大。

三向单词查找树将每个节点中的R条连接缩减到了3条，节约了大量空间，而查找效率只是略有下降。

5.2.3 与第3章中各种符号表实现的对比

  1. 对于字符串查找这样的操作，如果空间足够，那么R向单词查找树的速度是最快的，能够在常数次字符比较之内（如果key的长度为i，则最多比较i次）完成查找；
  2. 当空间不是很充裕时，三向单词查找树是最佳的选择，它需要对字符进行对数级别次数的比较。而第3章中的二叉查找树虽然也只需要对数级别次数的比较，但每次比较都是比较的字符串，相对三向单词查找树的比较字符，要慢一些；
  3. 除开R向单词查找树和三向单词查找树，散列表也是很有竞争力的，但它的功能不如单词查找树强大，要实现p474表中的某些字符串特殊操作比较困难。
